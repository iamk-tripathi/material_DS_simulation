# Tooltip

Tooltip provides concise supplemental guidance near a trigger. It is best for short clarifications that help users understand a control without permanently consuming space in the layout.

## Do's and Don'ts

<DosDonts
  dos={[
    'Use tooltips for supplemental guidance only',
    'Keep tooltip copy short and specific',
    'Attach the tooltip to a clear trigger',
    'Use rich tooltip only when the added treatment is justified',
    'Ensure the trigger still makes sense without the tooltip',
  ]}
  donts={[
    'Put essential instructions only in a tooltip',
    'Use tooltips for long-form content',
    'Show tooltips without a clear trigger relationship',
    'Rely on tooltip alone for accessibility labeling',
    'Use rich treatment for routine one-word hints',
  ]}
/>

## Accessibility

<DocSection title="Supplemental Use">
  Tooltip should add context, not carry essential meaning that users need in order to complete the task.
</DocSection>

<DocSection title="Assistive Technology">
  Hover and focus timing, dismissal behavior, and aria-describedby wiring are not visible in the provided material. Please confirm.
