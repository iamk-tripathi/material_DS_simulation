# Checkbox

Checkbox expresses a selection state through selected, indeterminate, and unselected visual treatments. It solves the need to show compact multi-select state with clear interaction feedback.

## Do's and Don'ts

<DosDonts
  dos={[
    'Use checkbox only for selection state',
    'Use indeterminate for partial group selection',
    'Pair checkbox state with a clear text label',
    'Use the error variant only for validation issues',
    'Preserve visible focus feedback for keyboard users',
  ]}
  donts={[
    'Use checkbox for mutually exclusive choices',
    'Use error styling without validation context',
    'Remove hover, focus, or pressed feedback',
    'Disable a checkbox without surrounding context',
    'Rely on color alone to convey selection state',
  ]}
/>

## Accessibility

<DocSection title="Contrast">
  Primary and error states use distinct semantic colors, but selection meaning should not rely on color alone.
</DocSection>

<DocSection title="Interaction States">
  Enabled, hovered, focused, pressed, and disabled states are all visible in the implementation.
</DocSection>
