# Navigation

Navigation presents top-level destination sets across bar, rail, drawer, and XR layouts. It solves the need to move between primary application sections while preserving a consistent destination model.

## Do's and Don'ts

<DosDonts
  dos={[
    'Use navigation only for top-level destinations',
    'Keep the destination set concise and stable',
    'Match the navigation variant to the layout context',
    'Preserve the active destination state clearly',
    'Use alignment intentionally on rail variants',
  ]}
  donts={[
    'Use navigation for secondary or temporary actions',
    'Overcrowd the destination set without hierarchy',
    'Mix bar, rail, and drawer patterns without a layout reason',
    'Hide the active destination state',
    'Use navigation labels that are vague or inconsistent',
  ]}
/>
