# Chip

Chip communicates compact input, assistive, filter, suggestion, and grouped choices in a small surface. It solves the need to show concise labels, lightweight actions, and filter states without taking up much space.

## Do's and Don'ts

<DosDonts
  dos={[
    'Use chip variants consistently within the same flow',
    'Keep chip labels short and scannable',
    'Use selected state to reinforce active filters',
    'Use grouped chips when multiple compact choices belong together',
    'Keep focus and drag feedback visible for interactive chips',
  ]}
  donts={[
    'Use chips as the primary call-to-action on a screen',
    'Mix too many chip styles without hierarchy',
    'Use long sentences inside chip labels',
    'Hide the selected state for active filter chips',
    'Rely on shadow alone to communicate chip emphasis',
  ]}
