# List

List presents structured rows for scanning, selection, disclosure, and swipe-driven actions. It is the core pattern for repeatable content with flexible leading and trailing anatomy.

## Do's and Don'ts

<DosDonts
  dos={[
    'Choose leading and trailing elements based on row meaning',
    'Use overline and supporting text only when they add hierarchy',
    'Use expandable rows for progressive disclosure',
    'Use dense lists intentionally in space-constrained layouts',
    'Keep swipe actions supplemental and discoverable',
  ]}
  donts={[
    'Mix many row patterns in the same list without reason',
    'Use list decoration that does not support the row action',
    'Hide essential actions only behind swipe',
    'Rely on selected background color alone to convey state',
    'Use dense rows when readability is the higher priority',
  ]}
/>
