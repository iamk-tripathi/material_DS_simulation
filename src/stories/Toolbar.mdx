# Toolbar

Toolbar groups related tools and quick actions inside a compact horizontal, vertical, or XR surface. It works best when a small set of high-frequency controls should stay together as one utility cluster.

## Do's and Don'ts

<DosDonts
  dos={[
    'Group only closely related tools together',
    'Use floating configuration only when elevation adds useful separation',
    'Match standard, vibrant, and XR treatments to the surrounding surface hierarchy',
    'Use toggleable actions only when an active mode needs to persist',
    'Keep toolbar actions short, recognizable, and high-frequency',
  ]}
  donts={[
    'Mix unrelated tools into one toolbar',
    'Use strong vibrant or XR treatment without hierarchy reasons',
    'Add low-priority content that competes with the core tools',
    'Hide unavailable state when controls are disabled',
    'Treat toolbar like a dumping ground for overflow actions',
  ]}
/>
