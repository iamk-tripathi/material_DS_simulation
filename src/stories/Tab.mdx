# Tab

Tab organizes peer destinations within the same context. It keeps related views close together so users can switch between them without leaving the current hierarchy.

## Do's and Don'ts

<DosDonts
  dos={[
    'Use tabs for closely related peer destinations',
    'Use fixed tabs for smaller peer sets',
    'Use scrollable tabs when the set exceeds the available width',
    'Prefer labels unless icon-only meaning is already obvious',
    'Preserve visible focus and selected indicators',
  ]}
  donts={[
    'Use tabs for unrelated destinations',
    'Hide important destination meaning behind ambiguous icons',
    'Use tabs when content should instead be separate pages',
    'Remove the selected indicator',
    'Rely on hover color alone to communicate current state',
  ]}
/>
