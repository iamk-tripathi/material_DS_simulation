# Card

Card groups related content and actions inside a single bounded surface. It solves the need to organize related information into a readable, self-contained container.

## Accessibility

<DocSection title="Keyboard Navigation">
  Interactive cards should expose a visible focus treatment and remain reachable through keyboard navigation.
</DocSection>

<DocSection title="Screen Reader">
  Card content should preserve a clear reading order and actionable elements should have descriptive labels.
</DocSection>

<DocSection title="Contrast">
  Card text should maintain clear contrast against the card surface and any focused outline should remain visually distinct.
</DocSection>
