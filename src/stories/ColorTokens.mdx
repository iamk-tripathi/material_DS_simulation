# Color Tokens

<DocSection title="Overview" description="Material Design 3 color tokens define both the tonal palette used to build a theme and the semantic roles consumed by components.">
  <p>Reference tokens describe the underlying tonal palettes. Semantic tokens describe the actual roles components should consume, such as primary, on-surface, or primary-container.</p>
  <ul>
    <li>Reference tokens: palette-building inputs</li>
    <li>Semantic tokens: component-facing roles</li>
    <li>Surface container tokens: elevation-aware surface hierarchy</li>
  </ul>
</DocSection>

<DocSection title="Usage Rules" description="Material Design 3 guidance for component token usage.">
  <p><strong>Source: Material Design 3 specification</strong></p>
  <ul>
    <li>Always use semantic tokens (md.sys.color.*) in components. Never use reference tokens (md.ref.palette.*) directly.</li>
    <li>Reference tokens are for building and customising themes only.</li>
    <li>Always pair a color role with its on- counterpart: primary background always uses on-primary for text/icons.</li>
    <li>Surface container hierarchy (lowest to highest elevation): surface-container-lowest -> surface-container-low -> surface-container -> surface-container-high -> surface-container-highest.</li>
    <li>Error tokens are reserved for destructive actions and validation states only. Never use for decorative purposes.</li>
    <li>Never use raw hex values in components - always reference theme tokens.</li>
  </ul>
</DocSection>
