# Dialog

Dialog interrupts the current flow with focused information or a small set of decisions. It is best for short, high-priority moments that need confirmation, selection, or acknowledgment without navigating away.

## Do's and Don'ts

<DosDonts
  dos={[
    'Keep the title and supporting copy short',
    'Use one clear primary action and one secondary action',
    'Use list dialogs for compact option sets',
    'Escalate to XR styling only when emphasis is truly needed',
    'Make cancel and confirm outcomes easy to understand',
  ]}
  donts={[
    'Stack multiple dialogs on top of each other',
    'Put long-form workflows inside a dialog',
    'Use a dialog when inline content would work better',
    'Overload the footer with many actions',
    'Use XR emphasis for routine low-priority confirmations',
  ]}
/>
